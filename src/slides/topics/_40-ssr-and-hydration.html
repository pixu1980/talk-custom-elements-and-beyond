SSR and Progressive Enhancement

Server first HTML, then enhance with customElements

Server Renders HTML

        <!-- Server output (no JS required to read) -->
<article>
  <h2>Latest News</h2>
  <ul>
    <li><a href="/news/1">Item 1</a></li>
    <li><a href="/news/2">Item 2</a></li>
  </ul>
</article>

Enhance when defined

        // Enhance progressively
class NewsList extends HTMLUListElement {
  connectedCallback(){
    this.classList.add('enhanced');
    // Optional: client-side sorting, filtering, etc.
  }
}

customElements.define('news-list', NewsList, { extends: 'ul' });

// Hydrate only if needed
customElements.whenDefined('news-list').then(() => {
  document.querySelectorAll('ul[is="news-list"]').forEach(el => el.classList.add('hydrated'));
});

Islands Architecture

        <!-- Static page with interactive islands -->
<main>
  <article>...static content...</article>
  <aside>
    <button is="counter-button">Count: 0</button>
  </aside>
</main>
        // Load only the island script
import('/components/counter-button.js');

SSR Tips

  • Do not rely on client init for critical content
  • Keep HTML semantic and accessible
  • Use whenDefined for safe hydration
  • Defer non critical islands
  • Measure hydration costs
